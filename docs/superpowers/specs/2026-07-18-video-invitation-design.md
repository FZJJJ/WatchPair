# WatchPair 同步视频邀请设计

## 目标

让房间内的一方将当前哔哩哔哩视频邀请给另一方。接收方无论正在浏览哪个网页，均可通过浏览器通知确认；确认后，当前活跃标签页跳转到同一个视频和分 P，并应用邀请发起时的播放进度、播放状态与倍速。

## 交互

1. 发起方在哔哩哔哩视频页、已进入房间时点击“同步视频”。
2. 插件发送视频 URL、视频标识和媒体状态给服务器，并转发给房间内另一人。
3. 接收方收到 Edge 通知，按钮为“前往观看”和“暂不跳转”。
4. 接收方确认后，插件将其当前活跃标签页导航到邀请 URL；视频内容脚本加载完成后应用保存的媒体状态。
5. 接收方拒绝、关闭通知或没有活跃标签页时，不导航且不改变房间状态。

## 架构

- 协议新增 `video-invitation` 客户端消息和服务端消息，包含 `invitationId`、标准化哔哩哔哩 URL、`VideoIdentity` 与 `MediaState`。
- 服务端只向房间中的另一条连接转发邀请，不将它写入周期性房间状态。
- 后台 Service Worker 收到邀请后使用 `chrome.notifications` 创建确认通知，并在内存中暂存邀请；通知点击事件找到最后聚焦窗口的活跃标签页并调用 `chrome.tabs.update`。
- 目标视频页的内容脚本在 `video-ready` 时通知后台；后台若有待应用邀请，则向该页单次发送 `media-operation`，随后清除待邀请。
- 弹窗只在当前 B 站视频已就绪且已加入房间时显示“同步视频”按钮。

## 权限与边界

- 新增 `notifications` 与 `tabs` 权限，用于显示确认通知和导航用户确认后的活跃标签页。
- 保持现有 `https://www.bilibili.com/video/*` host permission；插件不读取任意网页内容，也不向任意网页注入脚本。
- 仅允许跳转到经过解析和标准化的 `https://www.bilibili.com/video/BV...` URL。
- 每个浏览器实例只保留一个待确认邀请；新邀请覆盖旧邀请，通知关闭后清除。

## 验收

- 发起方不在房间或当前页面不是有效 B 站视频时，按钮不可用并显示原因。
- 接收方在任意网站点击通知“前往观看”后，活跃标签页跳转到邀请视频与分 P。
- 视频加载完成后，接收方进度、暂停状态和倍速匹配发起时状态。
- 点击“暂不跳转”、关闭通知或邀请过期不会导航。
- 既有播放、暂停、进度、重连和内容脚本打包测试继续通过。
